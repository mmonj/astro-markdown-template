# Starlight Starter Kit: Basics

This is a project is using the [Astro](https://astro.build/) web framework and the [Starlight](https://starlight.astro.build/) pagebuilder, designed for documentation sites built with Markdown.

Refer to Starlight's documentation for full details on authoring Markdown pages and details on Starlight's custom features: https://starlight.astro.build/guides/authoring-content/

This project is a template to get you started with Astro + Starlight. It includes example content and pre-set configurations.

**Project initialized via:**

```bash
npm create astro@latest -- --template starlight
```

## 🚀 Project Structure

Inside of your Astro + Starlight project, you'll see the following folders and files:

```text
.
├── public/
├── src/
│   ├── assets/
│   ├── content/
│   │   └── docs/
│   └── content.config.ts
├── astro.config.mjs
├── package.json
└── tsconfig.json
```

Starlight looks for `.md` or `.mdx` files in the `src/content/docs/` directory. Each file is exposed as a route based on its file name.

Images can be added to `src/assets/` and embedded in Markdown with a relative link.

Static assets, like favicons, can be placed in the `public/` directory.

## Sidebar Configuration

The sidebar navigation is configured in `astro.config.mjs` using the `sidebar` option. Starlight provides several approaches to organize your sidebar, each with different use cases:

### Approach 1: Autogenerated (Simplest)

Automatically generate sidebar entries for all `.md` and `.mdx` files in a directory, following your file tree's hierarchical structure and sorted alphabetically:

```javascript
export default defineConfig({
  ...
  ...
  integrations: [
    starlight({
      ...
      ...
      sidebar: [
        { label: "Notes", autogenerate: { directory: "notes" } },
        { label: "Reference", autogenerate: { directory: "reference" } },
      ],
      ...
      ...
    }),
  ],
});
```

This is the simplest approach—directories automatically become nested groups. All Markdown files and subdirectories in the folder are included automatically.

### Approach 2: Flat List of Items

Explicitly list specific pages as a flat list to curate which ones appear in the sidebar. The sidebar will also render as a flat list of items with no nesting, regardless of file structure:

```javascript
export default defineConfig({
  ...
  ...
  integrations: [
    starlight({
      ...
      ...
      sidebar: [
        {
          label: "Notes",
          items: [
            "notes",
            "notes/python-basics",
            "notes/python-basics/getting-started",
            "notes/python-basics/functions",
          ],
        },
      ],
      ...
      ...
    }),
  ],
});
```

Pages are listed in the order you specify, all appearing as direct children with no parent-child grouping. Only listed pages appear in the sidebar.

### Approach 3: Hierarchical with Nested Groups

Explicitly define nested groups with custom labels. This approach allows full control over hierarchy and ordering. Groups can be nested multiple levels deep.

```javascript
export default defineConfig({
  ...
  ...
  integrations: [
    starlight({
      ...
      ...
      sidebar: [
        {
          label: "Notes",
          items: [
            "notes",
            {
              label: "Python Basics",
              items: [
                "notes/python-basics",
                {
                  label: "Getting Started",
                  items: ["notes/python-basics/getting-started"],
                },
                {
                  label: "Advanced Topics",
                  items: ["notes/python-basics/functions", "notes/python-basics/debugging"],
                },
              ],
            },
          ],
        },
      ],
      ...
      ...
    }),
  ],
});
```

### Approach 4: Index-Only Auto-Discovery with Custom Plugin

This project uses a custom Starlight plugin (`indexOnlySidebarPlugin`) that automatically discovers and indexes **only** `index.md` files within specified directories. This is useful when your documentation structure relies on handrolled index files to present directory-level page structure.

**How it works:**

- Scans provided directories recursively for `index.md` files
- Extracts page titles from frontmatter `title` field for labels (or uses directory names if `dirnameDeterminesLabel: true`)
- Automatically creates nested groups based on directory hierarchy
- Sorts items consistently: page links before groups

**Configuration in `astro.config.mjs`:**

```javascript
import indexOnlySidebarPlugin from "./src/plugins/index-only-sidebar";

export default defineConfig({
  ...
  ...
  integrations: [
    starlight({
      ...
      ...
      plugins: [
        indexOnlySidebarPlugin({
          directories: [
            { label: "Reference", directory: "reference" },
            { label: "Algorithms", directory: "csci-323-algorithms" },
            {
              label: "Operating Systems",
              directory: "csci-340-operating-systems",
            },
            {
              label: "Algorithms for Big Data",
              directory: "csci-328-algorithms-for-big-data",
            },
          ],
        }),
      ],
      ...
      ...
    }),
  ],
});
```

**Label Generation:**

The `dirnameDeterminesLabel` option (defaults to `false`) controls how group labels and page titles are determined:

- **`dirnameDeterminesLabel: false` (default)** — Uses frontmatter `title` field for labels
  - Example: Directory `csci-316-kong` with `title: "CSCI 316 Principles of Programming Languages"` displays as "CSCI 316 Principles of Programming Languages"
  - Falls back to directory name if no `title` in frontmatter

- **`dirnameDeterminesLabel: true`** — Converts directory names to human-readable labels automatically
  - Example: Directory `csci-316-kong` displays as "CSCI 316 Kong"
  - Handles patterns like `csci-316` → `CSCI 316` (uppercases acronyms)
  - Converts hyphens to spaces and title-cases words

**Example structure with automatic result:**

Directory structure:

```text
src/content/docs/notes/
  ├── index.md
  ├── csci-316-kong/
  │   ├── index.md
  │   ├── announcements/
  │   │   └── index.md
  │   ├── slides/
  │   │   └── index.md
  │   └── reviews/
  │       ├── index.md
  │       └── exam1-review/
  │           ├── hard/
  │           │   └── index.md
  │           └── intermediate/
  │               └── index.md
```

Generated sidebar with `dirnameDeterminesLabel: true`:

```text
Notes
  ├── Notes (from "notes")
  └── CSCI 316 Kong (group, from "csci-316-kong")
      ├── CSCI 316 Kong (from "csci-316-kong")
      ├── Announcements (from "announcements")
      ├── Slides (from "slides")
      └── Reviews (group, from "reviews")
          ├── Reviews (from "reviews")
          ├── Hard (from "exam1-review/hard")
          └── Intermediate (from "exam1-review/intermediate")
```

**When to use this approach:**

- Your documentation relies on the organization of content existing in `index.md` files
- You want automatic discovery without manual configuration
- You prefer directory structure to determine sidebar hierarchy
- You want page titles pulled from frontmatter (use `dirnameDeterminesLabel: false`)
- You want consistent directory-derived labels without managing frontmatter (use `dirnameDeterminesLabel: true`)

### Notes on Page Visibility

- Pages **must** be included in the sidebar configuration to appear in navigation
- Pages are still indexed in search even if not in the sidebar—the sidebar only controls navigation visibility
- Use the frontmatter field `sidebar: { hidden: true }` to exclude a page from the sidebar while keeping it searchable
- Use `sidebar: { order: 1 }` in frontmatter to customize the sort order within autogenerated groups

## Frontmatter

Frontmatter is a block of YAML at the top of your Markdown files that provide metadata about the page. Starlight uses this metadata to control how pages are displayed and organized.

Example:

```text
---
title: This is the page title, also used as the primary header on the page
description: A brief description of the page for SEO purposes
---
```

Starlight markdown files support the following frontmatter fields:

| Field         | Type           | Description                                                                                                                                                               |
| :------------ | :------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `title`       | string         | **Required** Page title displayed in the browser and navigation. You may enclose the title value in quotes if it contains special characters, such as `:`, `[`, `]`, etc. |
| `description` | string         | Page description for SEO meta tags                                                                                                                                        |
| `draft`       | boolean        | Mark page as draft (not included in production build, so it's hidden from search and sidebars)                                                                            |
| `sidebar`     | object         | Control sidebar visibility: `{ hidden: true }` hides from sidebar                                                                                                         |
| `order`       | number         | Sort order in auto-generated sidebars (lower numbers appear first)                                                                                                        |
| `pagefind`    | boolean        | Whether to include this page in search results (default: `true`)                                                                                                          |
| `template`    | string         | Use a custom layout template                                                                                                                                              |
| `head`        | array          | Additional HTML to inject into the `<head>`                                                                                                                               |
| `lastUpdated` | date           | Override the last updated date                                                                                                                                            |
| `prev`        | object/boolean | Customize or disable the previous page link                                                                                                                               |
| `next`        | object/boolean | Customize or disable the next page link                                                                                                                                   |

## 🧞 Commands

All commands are run from the root of the project, from a terminal:

| Command              | Action                                           |
| :------------------- | :----------------------------------------------- |
| `npm install`        | Installs dependencies                            |
| `npm run dev`        | Starts local dev server at `localhost:4321`      |
| `npm run build`      | Build your production site to `./dist/`          |
| `npm run preview`    | Preview your build locally, before deploying     |
| `npm run astro ...`  | Run CLI commands like `astro add`, `astro check` |
| `npm run astro help` | Get help using the Astro CLI                     |
